# API ドキュメント（autodoc）作成計画

作成日: 2026-04-17

## 目標

`zig build` の autodoc 機能を使い、`zctx` ライブラリの API ドキュメントを HTML として生成・公開できる状態にする。

## 現状

- `build.zig` はモジュール定義（`b.addModule`）とテスト・example のステップのみ
- autodoc 生成ステップは未設定
- Zig バージョン: `0.16.0`（`mise.toml` より）

## 実装計画

### Phase 1: doc comment の追加

公開 API に `///` doc comment を付ける。対象ファイルと主要シンボル:

**`src/root.zig`**
- 1行目の `/// zctx: ...` を `//!` に変換してモジュールレベルコメントにする
- 再エクスポートしているシンボルへのコメント（`Signal`、`waitAny`、`Context`、`CancelError`、`OwnedContext`、`TypedKey`、`background`、`todo`、`cancelledContext`、`withCancel`、`withDeadline`、`withTimeout`、`withTypedValue`）

**`src/context.zig`**
- `CancelError`
- `Context` 型と `done()`、`err(io)`、`deadline()`、`typedValue()` メソッド
- `OwnedContext` 型と `cancel(io)`、`deinit(io)` メソッド
- `TypedKey`
- `background` / `todo` / `cancelledContext` 定数
- `withCancel(io, parent, alloc)`
- `withDeadline(io, parent, deadlineNs, alloc)`
- `withTimeout(io, parent, timeoutNs, alloc)`
- `withTypedValue(comptime Key, allocator, parent, val)`

**`src/signal.zig`**
- `Signal` 型と `isFired()`、`wait(io)`、`waitTimeout(io, ns)`、`fire(io)` メソッド
- `waitAny(io, signals)` 関数

### Phase 2: build.zig に autodoc ステップを追加

```zig
// build.zig に追加するコード（Zig 0.16.0 向け）
const lib = b.addStaticLibrary(.{
    .name = "zctx",
    .root_module = mod,
});

const docs = lib.getEmittedDocs();
const install_docs = b.addInstallDirectory(.{
    .source_dir = docs,
    .install_dir = .prefix,
    .install_subdir = "docs",
});

const docs_step = b.step("docs", "Generate API documentation");
docs_step.dependOn(&install_docs.step);
```

`zig build docs` で `zig-out/docs/` に HTML が生成される。

### Phase 3: mise タスクの追加

`mise.toml` に以下を追加する:

```toml
[tasks.build-docs]
description = "Build API documentation"
run = "zig build docs --summary all"

[tasks.serve-docs]
description = "Serve and open API documentation"
depends = ["build-docs"]
run = """
npx --yes serve zig-out/docs &
sleep 1
open http://localhost:3000
"""
```

- `depends = ["build-docs"]` でドキュメント生成を自動実行
- `npx --yes serve` は macOS 標準の Node.js を使用（mise で管理しない）
- `open` は macOS 標準コマンド

### Phase 4: ドキュメントの閲覧確認

```sh
mise run serve-docs
```

CORS 制約があるため、ファイルを直接開かずローカルサーバー経由で閲覧すること。

## 注意事項

- Zig 0.16.0 では `std.Io` の変更があるが、autodoc の `build.zig` API（`getEmittedDocs`、`addInstallDirectory`）は 0.15 から大きな変更はないはず。ビルドエラーが出た場合は `b.addStaticLibrary` の代わりに `b.addObject` を試す。
- doc comment は公開シンボル（`pub`）のみが autodoc に現れる。内部型（`WaiterNode` 等）は `pub` でなければ非表示。
- `//!` （コンテナレベルコメント）は `src/root.zig` の先頭に書く。
